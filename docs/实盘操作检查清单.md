# 科技股策略 v11.2 实盘操作检查清单

> 打印此清单，每日对照执行，确保操作规范

---

## 📅 每日检查清单

### ⏰ 开盘前 (8:30-9:25)

#### 系统检查
- [ ] 打开交易软件，确认账户正常
- [ ] 打开科技股信号系统
- [ ] 更新股票数据（如需要）

#### 信号检查
- [ ] 查看今日买入信号列表
- [ ] 确认信号强度 ≥ 85
- [ ] 检查信号股票是否在60只股票池内

#### 持仓检查
- [ ] 查看当前持仓数量（最多5只）
- [ ] 计算可用买入资金
- [ ] 检查是否有持仓需要卖出

#### 买入准备（如有信号）
- [ ] 确认买入金额 ≤ 总资金的12%
- [ ] 计算买入股数（100股整数倍）
- [ ] 设置好买入价格（开盘价附近）

---

### 🔔 开盘时段 (9:30-10:00)

#### 买入执行
- [ ] 按信号买入股票
- [ ] 确认成交价格和股数
- [ ] 在Excel"当前持仓"表添加记录
- [ ] 记录买入日期、价格、股数

#### 卖出执行（如触发条件）
- [ ] 执行卖出操作
- [ ] 确认成交价格
- [ ] 从"当前持仓"表删除记录
- [ ] 在"交易记录"表添加完整记录

---

### 📊 收盘后 (15:00-16:00)

#### 数据更新
- [ ] 更新所有持仓的"当前价格"
- [ ] 更新"最高价"（如果创新高）
- [ ] 检查持仓天数

#### 卖出条件检查

**止损检查（最重要！）**
- [ ] 当前价格 ≤ 止损价（买入价×0.955）？
- [ ] 如果是 → 明日开盘止损卖出

**止盈检查**
- [ ] 当前价格 ≥ 止盈价（买入价×1.20）？
- [ ] 如果是 → 明日开盘止盈卖出

**移动止盈检查**
- [ ] 最高盈利 ≥ 9%？
- [ ] 如果是，当前盈利比最高盈利回撤 ≥ 2.8%？
- [ ] 如果是 → 明日开盘移动止盈卖出

**RSI检查**
- [ ] RSI > 80？
- [ ] 如果是 → 明日开盘卖出

**趋势检查**
- [ ] MA5 < MA20 且当前亏损？
- [ ] 如果是 → 明日开盘卖出

**持仓时间检查**
- [ ] 持仓天数 ≥ 15天？
- [ ] 如果是 → 明日开盘卖出

#### 记录更新
- [ ] 保存Excel文件
- [ ] 备份数据（每周）

## 📋 买入检查清单

执行买入前，逐项确认：

| 检查项 | 条件 | 确认 |
|--------|------|------|
| 信号强度 | ≥ 85 | □ |
| 股票池 | 在60只池内 | □ |
| 当前持仓 | < 5只 | □ |
| 买入金额 | ≤ 总资金12% | □ |
| MA金叉 | MA5上穿MA20 | □ |
| 中期趋势 | 股价 > MA60 | □ |
| RSI范围 | 45 ~ 72 | □ |

**全部确认后方可买入！**

## 📋 卖出检查清单

触发任一条件即卖出：

| 卖出条件 | 触发规则 | 触发？ |
|----------|----------|--------|
| 止损 | 亏损 ≥ 4.5% | □ |
| 止盈 | 盈利 ≥ 20% | □ |
| 移动止盈 | 最高盈利≥9%，回撤2.8% | □ |
| RSI超买 | RSI > 80 | □ |
| 趋势反转 | MA5<MA20 且亏损 | □ |
| 持仓超时 | 持仓 ≥ 15天 | □ |

**触发任一条件 → 次日开盘卖出！**

## ⚠️ 红线规则（绝对不能违反）

### 🚫 禁止事项

1. **禁止不止损**
   - 亏损达到4.5%必须止损
   - 不要抱有侥幸心理
   - 不要等待反弹

2. **禁止超仓位**
   - 单只股票 ≤ 12%
   - 总持仓 ≤ 5只
   - 不要加仓摊低成本

3. **禁止追涨杀跌**
   - 只按信号买入
   - 不要追高买入
   - 不要恐慌卖出

4. **禁止频繁交易**
   - 按策略信号操作
   - 不要主观判断
   - 不要情绪化交易

5. **禁止借钱炒股**
   - 只用闲置资金
   - 不要融资融券
   - 不要影响生活

## 📞 紧急情况处理

### 大盘暴跌（跌幅>3%）
1. 不要恐慌
2. 检查持仓是否触发止损
3. 触发止损的股票次日卖出
4. 未触发止损的继续持有
5. 暂停新买入

### 个股跌停
1. 无法卖出时不要恐慌
2. 次日开盘挂跌停价卖出
3. 记录异常情况

### 系统故障
1. 使用备用交易渠道（手机APP）
2. 电话委托
3. 记录故障时间和影响
