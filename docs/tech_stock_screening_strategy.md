# 科技股筛选策略说明文档

## 概述

本文档详细说明科技股筛选系统的筛选策略、评分逻辑和决策依据，帮助用户理解系统的工作原理。

## 筛选策略框架

### 整体流程

```
全市场股票 (5000+)
       │
       ▼
┌─────────────────┐
│  第一层：基础过滤  │  排除ST、停牌、非主板
└─────────────────┘
       │ (~4500只)
       ▼
┌─────────────────┐
│  第二层：行业筛选  │  识别科技行业股票
└─────────────────┘
       │ (~800只)
       ▼
┌─────────────────┐
│  第三层：财务筛选  │  评估财务健康度
└─────────────────┘
       │ (~350只)
       ▼
┌─────────────────┐
│  第四层：市场筛选  │  评估流动性和稳定性
└─────────────────┘
       │ (~200只)
       ▼
┌─────────────────┐
│  第五层：综合评分  │  多维度加权评分

       │ (~100只)

```

## 第一层：基础过滤

### 过滤规则

| 规则 | 说明 | 原因 |
|------|------|------|
| 排除ST/*ST | 特别处理股票 | 财务异常或退市风险 |
| 排除停牌股 | 长期停牌股票 | 无法交易 |
| 仅保留主板 | 000/001/002/600/601/603/605 | 流动性和规范性更好 |
| 排除新股 | 上市不足60天 | 数据不足，波动大 |

### 代码示例

```python
# 主板股票代码模式
```

### 匹配算法

```python
def match_industry(name, business_desc, industry_name):
    """
    行业匹配算法
    
    1. 合并所有可用文本
    2. 遍历各行业关键词
    3. 统计匹配数量
    4. 计算置信度
    """
    text = f"{name} {business_desc} {industry_name}"
    
    best_industry = None
    best_score = 0
    
    for industry, keywords in INDUSTRY_KEYWORDS.items():
        matched = [kw for kw in keywords if kw in text]
        score = len(matched)
        
        # 名称中包含关键词加分
        if any(kw in name for kw in matched):
            score += 2
        
        if score > best_score:
            best_score = score
            best_industry = industry
    
    # 计算置信度
    confidence = min(1.0, best_score / 3)
    
    return best_industry, confidence
```

### 置信度阈值

| 置信度 | 说明 | 处理方式 |
|--------|------|----------|
| ≥ 0.7 | 高置信度 | 直接纳入 |
| 0.5-0.7 | 中置信度 | 纳入，标记待确认 |
| 0.3-0.5 | 低置信度 | 纳入，需人工审核 |
| < 0.3 | 不确定 | 排除 |

## 第三层：财务筛选

### 筛选标准

#### 盈利能力标准

| 指标 | 最低要求 | 优秀标准 | 权重 |
|------|----------|----------|------|
| ROE | ≥ 8% | ≥ 15% | 30% |
| ROA | ≥ 4% | ≥ 10% | 20% |
| 毛利率 | ≥ 20% | ≥ 40% | 25% |
| 净利率 | ≥ 5% | ≥ 15% | 25% |

#### 成长性标准

| 指标 | 最低要求 | 优秀标准 | 权重 |
|------|----------|----------|------|
| 3年营收CAGR | ≥ 10% | ≥ 25% | 30% |
| 3年利润CAGR | ≥ 15% | ≥ 35% | 35% |
| 研发投入占比 | ≥ 3% | ≥ 8% | 20% |
| 1年营收增长 | ≥ 5% | ≥ 20% | 15% |

#### 财务稳健性标准

| 指标 | 最高/最低要求 | 优秀标准 | 权重 |
|------|---------------|----------|------|
| 资产负债率 | ≤ 60% | ≤ 40% | 35% |
| 流动比率 | ≥ 1.2 | ≥ 2.0 | 25% |
| 速动比率 | ≥ 0.8 | ≥ 1.5 | 20% |
| 现金流量比率 | ≥ 0.1 | ≥ 0.3 | 20% |

#### 估值合理性标准

| 指标 | 最高要求 | 合理范围 | 权重 |
|------|----------|----------|------|
| PE | ≤ 50 | 15-35 | 35% |
| PB | ≤ 8 | 2-5 | 25% |
| PEG | ≤ 2.0 | 0.5-1.5 | 25% |
| PS | ≤ 10 | 2-6 | 15% |

### 评分算法

```python
def score_financial(indicators):
    """
    财务评分算法
    
    各维度独立评分后加权汇总
    """
    # 盈利能力评分
    profitability = (
        score_roe(indicators.roe) * 0.30 +
        score_roa(indicators.roa) * 0.20 +
        score_gross_margin(indicators.gross_margin) * 0.25 +
        score_net_margin(indicators.net_margin) * 0.25
    )
    
    # 成长性评分
    growth = (
        score_revenue_growth(indicators.revenue_growth_3y) * 0.30 +
        score_profit_growth(indicators.profit_growth_3y) * 0.35 +
        score_rd_ratio(indicators.rd_ratio) * 0.20 +
        score_revenue_growth(indicators.revenue_growth_1y) * 0.15
    )
    
    # 稳健性评分
    stability = (
        score_debt_ratio(indicators.debt_ratio) * 0.35 +
        score_current_ratio(indicators.current_ratio) * 0.25 +
        score_quick_ratio(indicators.quick_ratio) * 0.20 +
        score_cash_flow(indicators.cash_flow_ratio) * 0.20
    )
    
    # 估值评分
    valuation = (
        score_pe(indicators.pe_ratio) * 0.35 +
        score_pb(indicators.pb_ratio) * 0.25 +
        score_peg(indicators.peg_ratio) * 0.25 +
        score_ps(indicators.ps_ratio) * 0.15
    )
    
    return profitability, growth, stability, valuation
```

## 第四层：市场筛选

### 流动性标准

| 指标 | 最低要求 | 优秀标准 | 说明 |
|------|----------|----------|------|
| 总市值 | ≥ 50亿 | ≥ 200亿 | 确保公司规模 |
| 流通市值 | ≥ 30亿 | ≥ 100亿 | 确保可交易规模 |
| 日均成交额 | ≥ 0.5亿 | ≥ 2亿 | 确保交易活跃 |
| 换手率 | 0.5%-15% | 1%-5% | 避免过冷或过热 |

### 稳定性标准

| 指标 | 最高要求 | 优秀标准 | 说明 |
|------|----------|----------|------|
| 年化波动率 | ≤ 60% | ≤ 35% | 控制价格波动 |
| 最大回撤 | ≤ 50% | ≤ 25% | 控制下行风险 |
| 涨跌停频率 | ≤ 10% | ≤ 3% | 避免极端波动 |

### 流动性等级

| 等级 | 得分范围 | 特征 |
|------|----------|------|
| 优秀 | ≥ 85 | 大市值、高成交、适中换手 |
| 良好 | 70-84 | 中等市值、活跃交易 |
| 可接受 | 55-69 | 小市值但交易正常 |
| 较差 | 40-54 | 成交偏低 |
| 不足 | < 40 | 流动性严重不足 |

## 第五层：综合评分

### 评分权重

```
综合得分 = 财务健康度 × 35% 
         + 成长潜力 × 25% 
         + 市场表现 × 20% 
         + 竞争优势 × 20%
```

### 权重设计理由

| 维度 | 权重 | 理由 |
|------|------|------|
| 财务健康度 | 35% | 基本面是长期投资的基础 |
| 成长潜力 | 25% | 科技股核心价值在于成长 |
| 市场表现 | 20% | 流动性影响交易执行 |
| 竞争优势 | 20% | 护城河决定长期价值 |

### 定性评估

#### 竞争优势评估

| 因素 | 权重 | 评估方法 |
|------|------|----------|
| 行业地位 | 30% | 关键词匹配（龙头、领先、第一） |
| 技术壁垒 | 30% | 研发投入、专利数量 |
| 品牌优势 | 25% | 品牌知名度、客户粘性 |
| 管理质量 | 15% | 管理层稳定性、治理结构 |

### 评级标准

| 评级 | 分数 | 特征 |
|------|------|------|
| AAA | 90-100 | 各维度均优秀，行业龙头 |
| AA | 80-89 | 多数维度优秀，少数良好 |
| A | 70-79 | 整体良好，无明显短板 |
| BBB | 60-69 | 基本合格，有提升空间 |
| BB | 50-59 | 部分指标不达标 |
| B | 40-49 | 多项指标不达标 |
| C | < 40 | 整体较差 |

## 第六层：风险控制

### 集中度控制

| 维度 | 限制 | 说明 |
|------|------|------|
| 单一行业 | ≤ 25% | 避免行业风险集中 |
| 单只股票 | ≤ 5% | 避免个股风险集中 |
| 前5大持仓 | ≤ 40% | 确保分散度 |

### 风险评估

| 风险类型 | 评估指标 | 预警阈值 |
|----------|----------|----------|
| 集中度风险 | 行业占比 | > 25% |
| 流动性风险 | 日成交额 | < 0.3亿 |
| 波动性风险 | 年化波动率 | > 70% |
| 财务风险 | 负债率 | > 70% |

### 风险处置

| 风险等级 | 处置方式 |
|----------|----------|
| 低风险 | 正常纳入 |
| 中等风险 | 纳入并标记 |
| 高风险 | 限制权重 |
| 严重风险 | 排除 |

## 策略优化建议

### 市场环境适应

| 市场环境 | 调整建议 |
|----------|----------|
| 牛市 | 适当放宽估值标准，增加成长权重 |
| 熊市 | 提高财务稳健性要求，增加防御性 |
| 震荡市 | 平衡各维度权重，注重流动性 |

### 行业轮动应对

| 情况 | 调整建议 |
|------|----------|
| 新兴行业崛起 | 及时添加行业关键词 |
| 行业景气下行 | 提高该行业筛选标准 |
| 政策利好 | 适当增加相关行业权重 |
